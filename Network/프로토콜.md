## 프로토콜

```프로토콜(Protocol)``` 
> 네트워크에서 데이터를 주고받기 위해 네트워크를 구성하는 컴퓨터와 네트워크 장비들이 지켜야 할 규칙

* 전 세계의 수많은 네트워크가 연결된 인터넷에서는 서로 다른 벤더가 공급하는 다양한 컴퓨터와 네트워크 장비들이 연결된다.  
* 이때 서로 다른 장비들이 인터넷이라는 거대한 네트워크를 형성하기 위해서는 합의된 규칙이 필요한데, 이러한 규칙을 프로토콜이라고 한다.

### 1. 특징
* 프로토콜은 데이터 전송 상대방, 데이터 형식, 전송 순서, 절차, 전송 방법등을 규정
* 컴퓨터 간 데이터를 전송할 때는 다양한 프로토콜 중 서비스의 종류에 따라 프로토콜을 조합해서 사용
  * 웹서비스의 경우 TCP, IP, HTTP 프로토콜을 조합해서 사용
  * 같은 언어를 사용해야 대화가 가능한 것처럼 네트워크도 데이터 전송을 위해서는 컴퓨터와 네트워크 장비들이 같은 프로토콜 조합을 사용해야 함.

## 2. 기본 요소

통신하기 위한 약속인 프로토콜을 정의하기 위해서는 ```구문```, ```의미```, ```타이밍``` 3가지가 필요하다.

> ```구문``` : 데이터의 형식이나 부호화 및 신호 레벨을 규정한 것 
* 다른 매체와 어떤 단위로, 어떤 형식으로, 어떤 신호를 보낼지를 결정해야 한다. → 일종의 문법
* 이것을 바로 구문(Syntax)을 정해서 통신하기 전 미리 정의해야 한다.


> ```의미``` : 전송의 조작이나 오류 제어를 위한 제어 정보에 대한 규정
* 데이터의 각 항목이 무엇을 뜻하는지, 문장의 의미는 무엇인지와 같은 내용적 측면에서 각 매체가 이해할 수 있도록 결정
* 오류가 발생했을 때 어떤 식으로 제어할 것인지, 데이터의 속도가 한 쪽이 너무 빠를 때 어떤 식으로 흐름을 제어할 것인지 등 제어를 통해 원활한 소통을 도움

> ```타이밍``` : 접속되어 있는 개체 간의 통신 속도의 조정이나 메세지의 순서 제어 등을 규정
* 위에서 어떤 형식으로 어떻게 데이터를 제어할 것인지를 정했다면 데이터를 어떤 순서로, 어떤 속도로 전송할 것인지를 정해야 한다.


## 3. 기능

### 1. 단편화와 재조립

* 데이터의 용량이 매우 큰 경우 효율적인 전송을 위해서는 전송 가능한 일정 크기의 작은 데이터 블록으로 나누는 작업이 필요하다. → 단편화
* 이후, 송신 측에서 단편화된 데이터 블록을 수신측에서 재구성 하여 원래의 데이터로 복원한다 → 재조립
* 단편화된 데이터의 크기가 작을 수록 오류제어를 효율적으로 할 수 있으며, 트래픽 또한 감소한다.
* 하지만 데이터 블록을 너무 작게 단편화하면 그만큼 단편화와 재조립 시간이 많이 소요되므로 전송 효율이 떨어질 수 있다.

### 2. 캡슐화
 
* 상위 계층의 데이터에 각종 제어 정보 (헤더, 트레일러 등)를 추가하여 하위계층으로 내려보내는 과정
* 데이터가 어떤 주소를 가지고 있고, 오류는 어떻게 제어하고, 어떤 형식으로 보내는지 등에 대한 정보를 요약하여 가지고 있어야 한다.

### 3. 연결 제어
 
* 통신 개체간의 연결을 제어하며 연결 설정, 데이터 전송, 연결 해제 3단계로 구성
* 데이터를 주고 받는 시스템을 연결하여 안전하게 데이터를 전송하는 방법이 있고, 연결하지 않고 받든 말든 상관없이 빠르게 데이터를 전송하는 방법이 있는데 이러한 2가지 방법 중 하나를 결정해야 한다.

### 4. 오류 제어
 
* 전송하면서 발생하는 오류를 검출하고 정정하는 기능
* 데이터를 주고 받는 과정에서 오류가 발생했다면 이것을 확인할 수 있어야 하고, 해당 오류를 수정하고 손실되지 않도록 대비해야 한다.

### 5. 흐름 제어
 
* 데이터의 전송량이나 전송 속도 등을 제어
* 송수신 개체간의 처리 속도 차이를 조정하고, 수신 개체에서 처리할 수 있는 데이터 분량만큼 송신개체에 보내도록 함

### 6. 순서 제어
 
* 데이터를 어느 순서로 보낼지를 결정
* 이러한 순서는 오류가 발생했을 때 어느 지점에서 발생했는지 쉽게 확인할 수 있도록 한다.

### 7. 동기화
 
* 두 개의 통신 개체가 동시에 같은 상태를 유지하도록 하는 것 (즉, 타이밍을 맞추는 것)
* 데이터를 보내는 쪽과 데이터를 받는 쪽 둘 다 시작하는 시간, 검사하는 시간, 어떤 데이터 종류를 가지고 있는지 등 같은 상태를 유지해야 한다.

### 8. 주소 지정
 
* 송수신지의 주소를 부여함으로써 데이터의 정확한 전송을 보장한다.

### 9. 다중화
 
* 하나의 통신회선을 여러 통신 개체들이 동시에 접속할 수 있는 기술
* 여러 사용자들이 동시에 사용할 수 있도록 해야 한다.


## 종류

프로토콜 종류에 대한 간단한 정리

* ```HTTP(HyperText Transfer Protocol)``` : 웹(응용 계층) 상에서 하이퍼텍스트(HTML)를 통해 전송하는 규약
* ```FTP(File Transfer Protocol)``` : 응용 계층에서 파일을 주고 받기 위한 규약
* ```SSL(Secure Socket Layer)``` : 표현 계층의 네트워크의 인증, 암호화 등의 규약
* ```TCP(Transmission Control Protocol)``` :전송 계층에서 정보 전달을 안전하게 할 수 있도록 하는 규약
* ```UDP(User Datagram Protocol)``` :전송 계층에서 보다 빠르게 데이터를 주고 받을 수 있도록 하는 규약
* ```IP(Internet Protocol)``` : 네트워크 계층에서 호스트의 주소 지정과 패킷을 분할하고 조립하는 규약
